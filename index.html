JWT signing and calculating the number of days is performed in user model’s toJSON property, allowing it to run when necessary, without cluttering the rest of the code. Password hashing is handled with bcrypt in a pre save function, that runs only if the password was modified.   • Solutions

    A small npm package called clarifai-nodejs-grpc was used, along with the id for the necessary model, to set up the connection. One loop iterates over response outputs, and within each one, another loop iterates over it's regions, pushing any text into one main output array, which gets joined afterwards, with a next line, to avoid conflating text that actually should be separated. The app looks at the current hour, and based on it renders words like “morning”, “evening” etc, while the username is received in the props. There is a small loading animation while the image is being scanned, if an error occurs, it will be displayed, if an invalid link is pasted, it will say so instantly.

    Potential improvements

    Adding the ability to e-mail the converted text, adding more Clarifai models and with them more options, adding proper routing and more pages, adding a counter of converted items

  • Solutions

    The data from the contact form gets sent to a PHP file uploaded to the server, which then handles the mailing function. The react-i18next plugin combined with i18next-http-backend enables the site to draw text data straight from the locales folder, and replace it when necessary, without re-rendering. One language button is a link in disguise, which leads to a page that contains no other data, and, as such, won’t show a different language. The content for each page is outsourced to a separate component, which gets inserted into a “Layout” component (which contains the i18next provider) on every page.

    Potential improvements

    Adding the delayed support for the German version, adding a separate page about their machine park and services, adding the ability to order Air Guides automatically

  • Solutions

    BurgerPreview component receives one prop for size and two for seeds, and through them determines whether the burger is currently being built, and conditionally renders the necessary view. Food item page checks the URL for category and item variables, and inserts the category if it’s missing. The app uses reselect and exports custom selectors to avoid loading unnecessary state. JWTs are used and stored in local storage and verified in the backend when necessary. BCrypt is used to hash and securely store passwords. Redirects to the login page from guarded routes pass URLs to it, so after successful authentication you can proceed to the original route.

    Potential improvements

    Turning the orders page into a guarded admin dashboard with all data visible, adding online payment, adding and storing refresh tokens, adding user profile pages

    Live Code

  • Solutions

    Two different forms are connected to two different PHP files, making sure the mailing function does not rely on a third party. Just before an order gets sent, the product names are retrieved from a JSON file with identical IDs. The menu component receives a type prop, which drastically alters the look of each menu. The app uses react-router-sitemap-generator for creating a sitemap, and react-snap for pre-rendering content, improving it’s overall SEO. The .htaccess file renders index.html on every route without altering the url, which allows the React app to take over and load the proper page. In case of a route that actually doesn’t exist, the app loads a custom 404 page.

    Potential improvements

    Adding different pages for different product categories, adding the ability to order every product, adding profiles and authentication, adding online payment
